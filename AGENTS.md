# Agent Guidelines

## Purpose
This repository has been reset for a fresh planning cycle. Keep changes minimal, well-documented, and focused on delivering a playable retro-gaming MVP.

## Working Agreements
- Operate in English for code, comments, and documentation.
- Prioritise clarity and incremental commits over large, risky rewrites.
- Document every significant decision in the PR description.

## Code & Docs Expectations
- Prefer Markdown for planning artefacts until engineering foundations exist.
- When code is introduced, follow idiomatic TypeScript/React standards with strict linting.
- Include test plans or commands for any executable additions.

## Submission Checklist
- [ ] Files include relevant context or rationale.
- [ ] No secrets or credentials committed.
- [ ] Tests or validation steps are described when applicable.
